# CONTEXT_BUDGET.md

## 目标
控制启动上下文体积，保证执行速度与稳定性。

## 文件职责（单一职责）
- `AGENTS.md`：执行规则（启动/安全/长任务/验收）
- `SOUL.md`：人格、语气、边界
- `MEMORY.md`：长期有效结论与索引
- `TOOLS.md`：环境入口与低风险速查

## 预算（软上限）
- `AGENTS.md` ≤ 120 行
- `SOUL.md` ≤ 120 行
- `MEMORY.md` ≤ 100 行
- `TOOLS.md` ≤ 80 行

## 超预算处理顺序（必须按序执行）
1. 删重复：同一规则只保留一处“权威位置”
2. 改引用：把细节改为路径引用（`skills/`、`projects/`、`memory/`）
3. 迁内容：把过程型内容迁出核心文件
4. 再压缩：合并同义条目，保留结论句

## 写入规则
1. 新增前先问：是否长期有效？不是就不写核心文件
2. 优先“替换旧句”，避免净增长
3. 敏感信息（密钥/密码/令牌）不进入常驻上下文文件
4. SOP 细节放 skill/project，不塞到核心文件

## 巡检节奏
- 每周：检查四个核心文件是否超预算，超了当周处理
- 每月：做一次结构复盘，确认职责没有串位
