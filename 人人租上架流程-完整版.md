# 人人租上架流程 - 完整操作手册

> 给电子牛马兄弟的详细指南 🐂🐴

---

## 📋 流程总览

```
信息收集 → 图片准备 → 定价计算 → 后台填写 → 提交审核 → 验证确认
```

**预计耗时：** 单个商品 15-30 分钟

---

## 🎯 第1步：信息收集

### 必须确认的信息

| 信息 | 说明 | 示例 |
|------|------|------|
| 品牌 | 商品品牌 | 苹果、小米、华为 |
| 型号 | 完整型号 | Mac mini M4 / 小米平板5 Pro |
| 成色 | 新旧程度 | 90新、95新、全新 |
| 设备价值 | 闲鱼中位数（去除最低10%） | ¥3000 |
| 类目 | 平台类目路径 | 电脑/平板 > 平板 > 小米 |
| 商品图片 | 白底主图3-5张 | 本地路径或URL |

### 可选信息

- 功效描述（具体使用场景）
- 配置参数（内存/存储/屏幕等）
- 库存数量（默认1）

## ✍️ 第2步：标题生成

### 标题格式

```
{成色} {品牌} {型号} {功效描述}
```

### ⚠️ 标题规则（严格遵守）

1. **必须包含具体产品型号**
2. **功效描述必须是具体使用场景，不能是形容词**
3. **禁止词一律不出现**

### 功效描述示例

| 品类 | ❌ 错误 | ✅ 正确 |
|------|---------|---------|
| 电脑/主机 | 高性能主机 | 办公设计开发主机 |
| 平板 | 大屏平板 | 学习网课办公平板 |
| 校色仪 | 专业校色仪 | 摄影后期色彩校准校色仪 |
| 数位屏 | 高端数位屏 | 绘画设计手绘数位屏 |
| 投影仪 | 智能投影仪 | 家用观影办公投影仪 |

### 禁止词清单

| 禁止词 | 替换为 |
|--------|--------|
| 租赁 | 租用 |
| 出租 | 提供使用 |
| 免押/免息/分期 | （删除） |
| 最/最便宜/最低价 | （删除） |
| 测试 | 调试 |

## 🖼️ 第3步：图片准备

### 图片要求

- **尺寸：** 600×600px 及以上（推荐 800×800）
- **背景：** 纯白色 RGB(255,255,255)
- **数量：** 3-5张主图
- **第一张：** 必须为纯白底商品正面图
- **禁止：** 商家logo、水印、电话、微信等联系方式
- **布局：** 商品居中，四边适当留白

### 白底处理代码

```python
from PIL import Image
import numpy as np

img = Image.open('input.jpg')
arr = np.array(img)

# 阈值>200，覆盖浅灰色区域（JPEG压缩会导致灰化）
mask = np.all(arr[:,:,:3] > 200, axis=2)
arr[mask] = [255, 255, 255]

out = Image.fromarray(arr)
# 推荐PNG避免JPEG压缩灰化，如必须JPEG则quality=100
out.save('output.png')
```

### ⚠️ 图片验证

- 上传后必须下载服务器实际图片验证白底
- 人人租服务器会把PNG转JPG，JPEG压缩可能导致灰化
- 如发现灰化，降低阈值重新处理

## 💰 第4步：定价计算

### 使用定价脚本

```bash
python3 skills/rrz-publish/scripts/rrz_pricing.py <设备价值>
# 示例：python3 skills/rrz-publish/scripts/rrz_pricing.py 3000
```

### 定价规则

| 项目 | 公式 |
|------|------|
| 基础月租 | 设备价值 × 15% |
| 押金 | 设备价值 × 80% |

### 租期阶梯

| 租期 | 计算方式 |
|------|----------|
| 1天 | 月租/30 × 1.8 |
| 3天 | 月租/30 × 1.5 × 3 |
| 7天 | 月租 × 60% |
| 30天 | 月租 |
| 90天 | 月租×3 × 95折 |
| 180天 | 月租×6 × 9折 |
| 365天 | 月租×12 × 85折 |
| 730天 | 月租×24 × 80折 |
| 1095天 | 月租×36 × 75折 |

### 双方案定价

- **方案一（到期须归还）：** 基础价格
- **方案二（到期可归还/续租）：** 基础价格 × 1.15

### 运费规则

| 租期总租金 | 寄送 | 归还 |
|-----------|------|------|
| ≤¥300 | ❌ | ❌ |
| ¥301-500 | ✅ | ❌ |
| ¥501-800 | ✅ | ❌ |
| >¥800 | ✅ | ✅ |

### 5.5 商品信息（⚠️ 关键步骤）

#### 上传商品主图

1. 点击 `.add-btn-wrap`（+ 上传图片）
2. 弹出素材中心 → 上传文件
3. **关键：** 去掉file input的`webkitdirectory`属性
   ```javascript
   const input = document.querySelector('input[type="file"]');
   input.removeAttribute('webkitdirectory');
   input.removeAttribute('directory');
   ```
4. 用Playwright `set_input_files()` 上传
5. 等待上传完成 → 选中图片 → 确定

#### 上传描述图片（⚠️ 必须！）

1. 点击 **`.desc-add-btn-wrap`**（描述区域的上传按钮）
2. 同样流程上传图片
3. ⚠️ **不上传描述图片会导致"商品描述需要包含图片"错误！**
4. 后端检查的是 `detail_picture` 字段，不是 description 里的 `<img>` 标签

#### 设置商品描述

⚠️ **必须通过Vue实例设置，直接改DOM无效！**

```javascript
// 找到表单Vue实例
function findFormVue(el, depth) {
    if (depth > 10) return null;
    if (el.__vue__) {
        const data = el.__vue__.$data || {};
        if (data.formData && 'description' in data.formData) return el.__vue__;
    }
    for (const child of el.children) {
        const r = findFormVue(child, depth + 1);
        if (r) return r;
    }
    return null;
}

const vm = findFormVue(document.body, 0);
const fd = vm.$data.formData;

// 设置描述图片列表（映射为detail_picture）
fd.descPicList = [
    {url: 'https://img1.rrzuji.cn/...'},
    {url: 'https://img1.rrzuji.cn/...'}
];

// 设置描述HTML
fd.description = '<p>商品描述文字</p><img src="url1"/>';

// 同步编辑器UI
function findEditorVue(el, depth) {
    if (depth > 10) return null;
    if (el.__vue__) {
        const data = el.__vue__.$data || {};
        if ('editorContent' in data) return el.__vue__;
    }
    for (const child of el.children) {
        const r = findEditorVue(child, depth + 1);
        if (r) return r;
    }
    return null;
}
const editorVm = findEditorVue(document.body, 0);
if (editorVm) editorVm.$data.editorContent = fd.description;
const editor = document.querySelector('.w-e-text');
if (editor) editor.innerHTML = fd.description;
```

## ✅ 第6步：提交审核

### 提交方式

点击"提交审核"按钮，或通过API：

```
POST https://go-micro.rrzu.com/product/Spu/update
Body: { ...spu_data, status: 1 }
```

### ⚠️ 提交后验证（必须！）

1. 页面toast提示不可靠，必须实际验证
2. 通过API查询商品状态：`status: 1` = 审核中
3. 去"待审核"列表确认商品出现
4. 如果 `status: 0`（草稿），说明提交失败，需排查原因

---

## 🔍 第7步：审核跟踪

### 审核结果检查

- 定期检查商品列表中的审核状态
- 审核不通过 → 查看原因 → 按修复方案处理 → 重新提交

### 常见审核拒绝原因

| 原因 | 修复方式 |
|------|----------|
| 标题缺少功效描述 | 用具体使用场景词，不用形容词 |
| 标题型号与描述不一致 | 统一标题和描述中的型号 |
| 图片背景非纯白 | 降低白底处理阈值，用PNG格式 |
| 商品描述无图片 | 通过 `.desc-add-btn-wrap` 上传描述图片 |
| 包含"租赁"字眼 | 全文替换为"租用" |
| 包含敏感词"测试" | 替换为"调试" |
| 套餐名缺少租用方式 | 改为"租完归还"等标准名称 |
| 规格表头填写有误 | "套餐"改为"型号"或"配置" |
| 需要增加套餐规格 | 添加更多租期选项 |

## 💡 踩坑记录

### 图片上传

- 页面有两个上传区域：`.add-btn-wrap`（主图）和 `.desc-add-btn-wrap`（描述图片）
- 两个都要上传，缺描述图片会报错
- file input 默认有 `webkitdirectory` 属性，必须先移除

### 表单提交

- 必须按步骤顺序走完，不能跳步
- DOM修改不触发Vue双向绑定，必须通过Vue实例操作
- `descPicList` 前端字段名 → 提交时映射为 `detail_picture`

### 图片质量

- JPEG压缩导致白底灰化，推荐PNG
- 白底处理阈值用>200（不是>240）
- 上传后下载验证服务器实际图片
