## OpenClaw 运行时排障：后台 exec 超时误判
- 现象：`exec(background=true)` 跑长任务（如批量补图、3 分钟 smoke test）时，会在一段时间后收到 `SIGTERM`，表面看像 heartbeat / cron 抢断。
- 实验结论：核心根因不是 heartbeat，也不是单个 cron；而是 **后台 exec 仍受 `timeout` 约束**。对照实验：
  - `sleep 70` + `background=true` + `timeout=30` → 被 `SIGTERM`
  - `sleep 70` + `background=true` + `timeout=120` → 同时间窗仍存活
- 教训：以后长任务不要再默认配短 timeout；`background=true` 也不能假定会绕过 timeout。
- 暂定策略：
  1. 短任务保留短 timeout
  2. 后台/批量任务默认提高到 10~15 分钟以上
  3. 长任务分批并落盘，避免整包长跑
- 系统缺陷：超时后缺少明确提示、后台任务缺少登记/监管、用户与 agent 都难知道任务是“还在跑”还是“已被杀”。
- 已建立方案文档：`projects/openclaw-timeout-observability-plan.md`
