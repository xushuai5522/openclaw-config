## 技术要点

### OpenClaw远程配置管理
- **配置文件**: `~/.openclaw/openclaw.json`
- **修改方式**: 使用jq命令行工具修改JSON配置
- **重启命令**: `openclaw gateway restart`
- **验证方式**: 重启后检查日志确认配置生效

### 模型配置教训
- **错误做法**: 自作主张添加前缀后缀（如 `claude-once/aws.amazon/claude-opus-4-6:once`）
- **正确做法**: 严格按大哥要求，不加不减（如 `claude-opus-4-6`）
- **原则**: 配置修改必须精确执行，不要自行推测

### SSH远程操作
- **工具**: sshpass（用于自动输入密码）
- **命令格式**: `sshpass -p 'PASSWORD' ssh user@host 'COMMAND'`
- **超时设置**: 使用 `timeout` 参数避免长时间卡死
- **问题**: 连接不稳定，需要考虑重试机制

## 教训总结

1. **配置修改要精确**: 不要自作主张添加或修改配置内容，严格按要求执行
2. **SSH连接不稳定**: 需要增加超时处理和重试机制，避免长时间等待
3. **验证要充分**: 配置修改后必须验证是否真正生效，不能只看重启成功
4. **沟通要及时**: 遇到问题及时汇报，不要自己猜测解决方案
