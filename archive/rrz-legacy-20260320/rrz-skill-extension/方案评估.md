# 人人租后台自动化方案评估

## 方案对比

### 方案A：Chrome插件
**优势：**
- 功能强大，可以使用完整的Chrome API
- 可以持久化存储配置
- 可以添加右键菜单、工具栏按钮等UI
- 可以跨页面通信
- 更专业的用户体验

**劣势：**
- 需要打包、安装、更新流程
- 开发调试相对复杂
- 需要manifest配置
- 分发需要Chrome Web Store或手动安装

### 方案B：油猴脚本（Tampermonkey/Violentmonkey）
**优势：**
- 轻量级，一个.js文件搞定
- 安装简单，直接导入即可
- 调试方便，修改即生效
- 分发容易，复制粘贴就行
- 用户基数大，门槛低

**劣势：**
- 功能受限于GM API
- UI集成相对简陋
- 存储依赖GM_setValue/GM_getValue

## 最终选择：油猴脚本 ✅

**理由：**
1. **快速迭代**：人人租后台可能会更新，油猴脚本修改更灵活
2. **易于分发**：大哥可以直接复制给其他人用
3. **调试友好**：出问题直接改代码，不用重新打包
4. **功能够用**：核心需求（DOM操作、表单填充、图片上传）油猴完全能实现
5. **兼容性好**：不依赖Chrome特定API，Firefox/Edge也能用

## 技术方案

### 核心功能模块
1. **Vue实例访问**：通过`__vue__`或`$vm`访问组件数据
2. **表单自动填充**：读取配置JSON，自动填充所有字段
3. **图片批量上传**：模拟文件选择和上传流程
4. **配置管理**：GM_setValue存储常用配置模板
5. **UI面板**：悬浮按钮+配置面板

### 兼容性策略
- 使用选择器时优先用data-*属性，其次class，避免依赖id
- 监听DOM变化（MutationObserver）适配动态加载
- 版本检测：记录后台版本特征，提示用户更新脚本
