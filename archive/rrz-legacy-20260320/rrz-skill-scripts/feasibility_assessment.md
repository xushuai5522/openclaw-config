# 混合方案可行性评估报告

## 执行摘要

混合方案（浏览器自动化 + API调用）在技术上**可行但有条件限制**。核心依赖是**方案1的API逆向结果**，如果API逆向失败，混合方案将降级为纯浏览器方案。

**评估结论：⭐⭐⭐⭐☆ (4/5) - 推荐实施，但需先完成API逆向**

## 1. 技术可行性分析

### 1.1 已验证的技术点 ✅

| 技术点 | 状态 | 证据 |
|--------|------|------|
| CDP浏览器连接 | ✅ 已验证 | `rrz_helper.py`成功连接 |

### 1.2 待验证的技术点 ⚠️

| 技术点 | 风险等级 | 验证方法 |
|--------|----------|----------|
| 图片上传API | 🔴 高 | 需抓包分析 |
| 方案配置API | 🔴 高 | 需抓包分析 |
| 认证机制 | 🟡 中 | 测试token有效期 |
| 反爬虫策略 | 🟡 中 | 监控请求频率限制 |

### 1.3 技术难点

**难点1：API接口发现**
- 人人租后台是SPA应用，接口可能混淆
- 需要监听网络请求，逆向分析
- 解决方案：已实现`discover-api`命令

**难点2：认证信息传递**
- 需要从浏览器提取token/cookies
- API请求需要正确的headers
- 解决方案：`RRZSession.extract_auth()`

**难点3：错误处理**
- API失败时需要降级到浏览器
- 需要保存现场（截图、日志）
- 解决方案：try-except + 降级机制

## 2. 性能提升预估

### 2.1 纯浏览器方案 vs 混合方案

| 操作 | 浏览器耗时 | 混合方案耗时 | 提升 |
|------|-----------|-------------|------|
| 登录 | 5s | 5s | - |
| 填基础信息 | 10s | 10s | - |
| 上传5张图 | 50s | 15s | **70%** ⬇️ |
| 添加3个方案 | 30s | 5s | **83%** ⬇️ |
| 提交审核 | 5s | 5s | - |
| **总计** | **100s** | **40s** | **60%** ⬇️ |

**结论：混合方案可将发布时间从100秒降到40秒**

### 2.2 并发优化潜力

```python
# 串行上传（浏览器）
for img in images:
    upload(img)  # 5张 × 10s = 50s

# 并发上传（API）
await asyncio.gather(*[upload(img) for img in images])  # 15s
```
