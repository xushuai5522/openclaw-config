# 人人租运营工具箱

> 职责：后台基础操作（登录、弹窗、审核修复、列表查询）。
> 发布规则/定价 → `rrz-publish`；完整上架流程 → `rrz-e2e`。

## 基础信息

| 项目 | 值 |
|------|-----|
| 后台地址 | https://admin.rrzu.com |
| 商品列表 | https://admin.rrzu.com/spu-view/list |
| 发布页 | https://admin.rrzu.com/spu-view/create |
| 编辑页 | https://admin.rrzu.com/spu-view/create?id={id} |
| API 基础 | https://go-micro.rrzu.com |
| iframe | name=rrzuji, 域名 admin-vue.rrzu.com |
| 店铺 | 书电马数码店 |

## 页面结构
- 主页面 `admin.rrzu.com` + iframe `admin-vue.rrzu.com`
- JS 注入需通过 `frame: "rrzuji"` 参数

## 弹窗处理
1. 经营地址信息补充 → 点“跳过”
2. 方案应用邀请 → 点关闭（不点“确定”）
3. 工单提醒 → 点“×”
4. 通知面板 → 点 Close

## 登录恢复
1. 访问 `https://www.rrzu.com/auth/server-login`
2. 切到“密码登录”
3. 输入手机号 + 密码
4. 勾选隐私协议
5. 点击登录
6. 等待跳转到 `admin.rrzu.com`

## 审核规则速查（后台修复口径）

### 标题
- 格式：`{成色} {品牌} {型号} {使用场景}`
- 必须含具体型号
- 标题型号必须与描述/商品实际型号一致

### 禁表述
- `租赁` → `租用`
- `出租` / `免押` / `免息` / `分期` / `最` → 删除或改写
- `测试` → `调试`

### 套餐
- 套餐名称必须使用租用方式模板：`租完归还`、`可归还`、`到期可归还/续租`、`随租随还`
- 禁止使用：`套餐1`、`标准版`、`方案A` 这类泛化命名
- 规格表头用“型号/参数/配置”，不用“套餐”
- 若因规格不足被打回，应补足到至少两种有效租用方式

### 图片
- 主图 + 描述图都必须有
- 商品描述必须包含图片
- 当前统一执行：白底 `#FFF`，尺寸 `≥600×600`（推荐 `800×800`）

## 审核修复流程
1. 商品列表找到“审核不通过”商品
2. 点“修改商品”进入编辑页
3. 按不通过原因修复（标题/套餐名/禁词/规格表头/描述图/型号一致性）
4. 提交审核
5. 回列表确认状态变化（不信 toast）

### 常见不通过原因

| 原因 | 修复 |
|------|------|
| 标题缺型号 | 添加完整品牌+型号 |
| 标题与描述型号不一致 | 统一型号写法 |
| 套餐名缺租用方式 | 改为标准租用方式模板 |
| 套餐内未说明租用方式 | 在套餐名/内容中明确写清 |
| 规格表头有误 | “套餐”→“型号/参数/配置” |
| 需增加套餐规格 | 补至少一个有效规格，当前默认补到两种租用方式 |
| 含“租赁”等禁表述 | 全量替换/删除 |
| 商品描述无图片 | 上传描述图 |

---

## 防重复创建
1. 新建前先搜索列表查重
2. `/spu-view/create` 无 id = 新建；有 id = 编辑
3. 提交后必须回列表拿到商品 ID + 状态变化
4. 状态未知时禁止二次新建
