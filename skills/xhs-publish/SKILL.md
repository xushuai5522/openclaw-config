## 发布流程

### 完整流程
```
素材收集 → 素材筛选 → 文案撰写 → 图片处理 → 上传发布 → 数据跟踪
```

### 第1步：素材收集

#### 来源
- 飞书群聊：大哥/家人发的照片视频
- NAS：家庭NAS（待接入）

#### 飞书图片下载方法
1. 获取 tenant_access_token
2. 通过消息列表API获取 image 消息的 message_id 和 image_key
3. 下载：`GET /im/v1/messages/{msg_id}/resources/{img_key}?type=image`
4. ⚠️ 合并转发消息中的 image_key 和原始消息不同

#### 素材存储
```
/Users/xs/.openclaw/workspace/xiaohongshu/
├── 素材/          # 原始照片
├── 缩略图/        # 压缩预览
└── 已发布/        # 已使用的素材归档
```

#### 规则
- ⚠️ 群内发的所有照片系统都要保存原图
- 运营过程要做好记录和记忆

### 第2步：素材筛选

#### 选图原则
- 高清、自然光、不过度滤镜
- 宝宝表情生动（笑、好奇、搞怪优先）
- 场景多样化（室内/室外/吃饭/玩耍）
- 封面图选最抓眼球的那张

#### 图片处理
- 超过2MB的需要先压缩：`sips -s formatOptions 70 -Z 2000 input.jpg`
- 不要过度修图，保持真实感

### 第3步：文案撰写

#### ⚠️ 文案风格（核心规则！）
- **拟人、俏皮、口语化**
- **避免AI味！** 不要用"在这个...的日子里"、"每一个瞬间都值得珍藏"这类套话
- 像真人妈妈/爸爸随手写的，不是文案策划写的
- 带点小幽默、小吐槽
- 短句为主，别写长段落

#### 标题公式
- 带具体年龄 + 痛点/好奇心/情绪
- ✅ "12个月的她，已经学会用眼神拿捏爸爸了"
- ✅ "女儿1岁了！这小表情也太搞笑了吧"
- ❌ "今天带宝宝出去玩了"（太平淡）
- ❌ "记录宝宝美好的一天"（AI味重）

#### 正文结构
```
开头：1-2句抓注意力（搞笑/共鸣/悬念）
中间：3-5个要点（用emoji分隔，短句）
结尾：引导互动（"你家宝宝也这样吗？"）
标签：5-8个相关话题标签
```

#### 标签库
```
#萌娃日常 #宝宝日常 #女儿奴 #育儿日记 #带娃日常
#1岁宝宝 #12个月宝宝 #宝宝成长记录 #亲子日常
#新手爸妈 #奶爸日常 #爸爸带娃 #宝宝辅食
#育儿分享 #育儿干货 #萌娃 #可爱宝宝
```

### 第4步：上传发布

#### 发布方式
- 通过小红书网页版（浏览器自动化）
- 图片上传：用 `ref=e120`（Choose File按钮）上传成功率高

#### 发布规则
- ⚠️ **默认只保存草稿，由大哥手动点发布**
- 特殊情况（大哥明确说可以直接发）才自动发布
- 发布前必须截图确认内容无误

#### 发布时间
- 最佳时段：晚上 19:00-22:00（母婴用户活跃高峰）
- 次优时段：中午 12:00-13:00

#### 发布频率
- 每周 3-5 篇（稳定更新比爆更重要）
- 图文和视频交替发

### 第5步：数据跟踪

#### 每周复盘
- 浏览量、互动率（点赞/收藏/评论）、涨粉数
- 爆款分析：哪篇效果好，为什么
- 下周内容规划调整

## 注意事项

### 隐私保护
- ⚠️ 避免暴露家庭住址、幼儿园等敏感信息
- 宝宝正脸照片需大哥确认可以发

### 平台规范
- 不能引导站外交易
- 不能刷数据
- 不要过度商业化，前期以真实分享为主

### 踩坑记录
- 第一条笔记因AI味太重被大哥要求删除重写
- 文案必须口语化、俏皮，像真人写的
- 图片上传用 Choose File 按钮（ref=e120）成功率高
- 标签用正文末尾 # 号添加，不要用系统推荐标签（会插到正文开头）
