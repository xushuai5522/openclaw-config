# Second Brain — 知识管理

## 记忆架构

```
workspace/
├── MEMORY.md          ← 长期记忆（每次主会话加载，精炼核心知识）
├── TOOLS.md           ← 工具和环境速查
├── memory/
│   ├── YYYY-MM-DD.md  ← 日常原始记录
│   ├── code-architecture.md  ← ESP32 模组架构
│   ├── mcu-architecture.md   ← MCU 仓库架构
│   ├── code-risks.md         ← 代码安全隐患
│   ├── bug-patterns/         ← 缺陷知识库（按项目）
│   ├── bug-analysis-docs.md  ← 分析文档索引
│   ├── port-services.md      ← 服务端口清单
│   ├── evolver-log.md        ← 进化日志
│   └── ...
```

## 记忆层次

| 层 | 文件 | 更新频率 | 加载时机 |
|----|------|---------|---------|
| L1 热记忆 | MEMORY.md | 每周精炼 | 每次主会话 |
| L2 日记忆 | memory/YYYY-MM-DD.md | 每日 | 当天+昨天 |
| L3 领域知识 | memory/*.md | 按需 | memory_search 检索 |
| L4 原始数据 | bug-patterns/, 日志 | 分析时生成 | 仅在分析时读取 |

## 操作规范

### 写入规则
- **MEMORY.md 只放精炼知识** — 不放原始日志、不放临时信息
- **日文件记录原始事件** — 决策、对话要点、发现
- **领域文件按主题组织** — 代码架构、Bug 模式、服务清单
- **每条记忆标注来源** — 从哪次对话/分析中学到的

### 维护任务（每周一次）
1. 检查近 7 天日文件，提取值得长期保留的知识
2. 更新 MEMORY.md（增加新知识、删除过时信息）
3. 检查知识库索引是否完整
4. 清理超过 30 天的日文件中的低价值内容

### 检索规则
1. 先 memory_search → 看相关片段
2. 再 memory_get 定位精确内容
3. 不确定时说"我查了记忆但没找到"

## MEMORY.md 章节规范

```
# MEMORY.md
## 关于大哥（个人偏好、工作习惯）
## 协作人员（姓名、open_id、角色）
## 技术知识（嵌入式核心知识、踩坑总结）
## 飞书 API 经验（接口踩坑、Block 类型）
## Coding API 经验
## 工作流经验（文档规范、Bug 分析流程）
## 代码仓库架构（索引，详情在领域文件）
## 知识库索引（所有 memory/ 文件清单）
## 自我进化教训（回答规范、工作习惯）
```

## 安全
- 主会话才加载 MEMORY.md（含个人信息）
- 群聊/共享场景不加载
- 不在消息中泄露 MEMORY.md 的私人内容
