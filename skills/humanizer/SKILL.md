# Humanizer - 去除AI味

## 快速参考

- **触发词**: 去AI味、人性化、改得自然点、像人写的
- **核心功能**: 识别AI写作模式、优化文案风格
- **适用场景**: 小红书文案、营销文案、日常对话

---

## 使用示例

### 示例1: 小红书宝宝文案优化

**AI原文（满满的AI味）**:
```
在这个充满阳光的午后，我们的小宝贝迎来了她人生中的第一次户外探索之旅。
值得注意的是，宝宝展现出了惊人的好奇心和探索欲。
每一个瞬间都值得珍藏，每一次成长都令人感动。
总之，这是一次非常有意义的体验，让我们深刻感受到了陪伴的重要性。
#宝宝成长记录 #珍贵时光
```

**优化后（自然人话）**:
```
今天带她去公园，小家伙看到草地就兴奋得不行。
爬了两步就坐下抓草，抓完还往嘴里塞😂
虽然全程就爬了不到5米，但她自己玩得可开心了。
回家路上累得直接睡着了，小手还握着一根草。
#12个月宝宝 #第一次户外
```

**优化要点**:
- ❌ 删除："在这个...的午后"、"值得注意的是"、"总之"
- ✅ 添加：具体细节、真实情绪、小吐槽
- ✅ 语气：口语化、短句、带emoji

### 示例2: 产品文案优化

**AI原文（营销味太重）**:
```
这款产品采用了业界领先的技术，为用户带来了前所未有的体验。
从多个维度来看，它都是市场上的佼佼者。
首先，在性能方面表现卓越；其次，设计理念独具匠心；最后，性价比极高。
总而言之，这是一款不容错过的优质产品，强烈推荐！
```

**优化后（真实推荐）**:
```
用了一周，说说真实感受。
性能确实不错，日常用完全够。设计也挺好看的，拿手里有质感。
价格比同类产品便宜200块，这点挺实在。
如果你预算有限又想要点品质，可以考虑。
```

**优化要点**:
- ❌ 删除："业界领先"、"前所未有"、"首先其次最后"、"强烈推荐"
- ✅ 添加：真实使用感受、具体对比、适用人群
- ✅ 语气：客观、不夸张、给建议而非推销

### 示例3: 工作汇报优化

**AI原文（过度正式）**:
```
尊敬的领导：
非常感谢您在百忙之中抽出宝贵时间阅读本报告。
关于本周工作进展，我将从以下三个方面进行详细汇报：
首先，在项目推进方面，我们取得了显著成效...
其次，在团队协作方面，我们建立了良好的沟通机制...
最后，在问题解决方面，我们采取了积极有效的措施...
综上所述，本周工作总体进展顺利，但仍需继续努力。
```

**优化后（简洁专业）**:
```
本周工作进展：

1. 项目推进：完成了3个核心功能，比计划提前2天
2. 团队协作：建立了每日站会机制，沟通效率提升
3. 遇到的问题：第三方API不稳定，已联系供应商解决

下周计划：完成剩余2个功能，准备测试环境。
```

**优化要点**:
- ❌ 删除："尊敬的"、"百忙之中"、"宝贵时间"、"综上所述"
- ✅ 改为：直接说重点、用数据说话、列出具体行动
- ✅ 结构：清晰的列表，不用"首先其次最后"

### 示例4: 技术文档优化

**AI原文（过度解释）**:
```
在深入探讨这个问题之前，我们首先需要理解其背后的技术原理。
从某种意义上来说，这个方案在多个维度上都具有显著优势。
值得注意的是，我们需要特别关注以下几个关键点...
总的来说，这是一个相对复杂但非常有价值的技术方案。
```

**优化后（直接清晰）**:
```
技术原理：使用Redis缓存减少数据库查询。

优势：
- 响应速度提升80%
- 数据库负载降低60%
- 支持高并发访问

注意事项：
- 缓存过期时间设置为5分钟
- 需要处理缓存穿透问题
```

**优化要点**:
- ❌ 删除："在深入探讨之前"、"从某种意义上"、"值得注意的是"
- ✅ 改为：直接说原理、用数据说话、列出要点
- ✅ 结构：标题+列表，一目了然

## 技能说明

这个技能可以将 AI 生成的文字处理得更像人写的，去除 AI 味、陈词滥调和过度营销语言。

## 使用场景

1. **生成回复前**：检查并优化自己的输出
2. **用户要求优化**：大哥说"去 AI 味"或"人性化一点"
3. **处理外部内容**：需要把 AI 生成的文案改得更自然

## 执行步骤

### 第一步：识别 AI 味特征

检查以下常见 AI 写作模式：
- 过度使用"首先、其次、最后"
- 机械的过渡句"值得注意的是..."
- 夸张的营销词汇"太棒了"、"绝绝子"
- 过度礼貌的套话"非常感谢您的..."
- 机械的总结句"总之..."
- 程式化的开头"下面我将从X个方面..."

### 第二步：优化策略

1. **删除过渡性废话**
   - 原文："值得注意的是，我们需要注意的是..."
   - 改："需要..."

2. **替换机械表达**
   - 原文："我认为从某种意义上来说"
   - 改："我觉得"

3. **去除夸张词汇**
   - 原文："太棒了"、"绝绝子"、"YYDS"
   - 改：正常使用词汇

4. **简化复杂句式**
   - 原文："关于这个问题，我们需要从多个维度进行深入分析和探讨"
   - 改："这个问题需要分析"

5. **保留必要连接词**
   - 真正需要逻辑连接的地方保留
   - 去掉为了"看起来有逻辑"而加的连接词

### 第三步：输出优化结果

优化后询问用户是否满意，如需调整可以进行二次优化。

## 注意事项

- 不要过度简化导致丢失重要信息
- 保留技术性内容的准确性
- 适度保留个人风格
- 核心观点不能变

## 成功标准

- 文字更自然流畅
- 去除明显 AI 味
- 用户满意
