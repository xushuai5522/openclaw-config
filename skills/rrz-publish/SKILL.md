# 人人租商品发布规则

> 职责：发布规则、定价计算、表单填写规范。
> 后台基础操作 → `rrz`；完整上架流程 → `rrz-e2e`。
>
> **重要：本技能强制分两层。**
> - **平台硬规则层**：来自《人人租商品发布规范》、平台审核驳回原文、平台真实校验提示。
> - **内部定价策略层**：是我方运营策略，不得冒充平台明文规则。

---

## 1. 平台硬规则层（提审前必须满足）

### 1.1 标题规则
标题格式：`{成色} {品牌} {型号} {具体使用场景}`

强约束：
1. **必须含具体型号**，不能只写品类或泛称。
2. **必须带具体使用场景**，场景优先用动词短语，避免空泛形容词。
3. **标题型号、描述型号、商品实际型号必须一致**。

场景示例：

| 品类 | ❌ 错误 | ✅ 正确 |
|------|---------|---------|
| 电脑/主机 | 高性能主机 | 办公设计开发主机 |
| 平板 | 大屏平板 | 学习网课办公平板 |
| 校色仪 | 专业校色仪 | 摄影后期色彩校准校色仪 |
| 数位屏 | 高端数位屏 | 绘画设计手绘数位屏 |
| 投影仪 | 智能投影仪 | 家用观影办公投影仪 |

### 1.2 禁表述
已确认高频禁表述：
- `租赁` → 改 `租用`
- `出租` → 删除/改写
- `免押` / `免息` / `分期` / `最` → 删除
- `测试` → 改 `调试`

执行要求：
- 标题、套餐名、卖点文案、描述图文案都要扫一遍。
- 若出现新驳回词，先补 `projects/rrz-platform-rules.md`，再补本技能。

### 1.3 套餐与规格规则
1. **套餐名必须是租用方式模板**，不得自造含糊名称。
2. 当前已确认可用模板：
   - `租完归还`
   - `可归还`
   - `到期可归还/续租`
   - `随租随还`
3. **套餐内需明确具体租用方式**，不能只写“套餐1/套餐2”。
4. **规格表头不能写“套餐”**，应改为：`型号` / `参数` / `配置`。
5. 若平台提示“请增加一个套餐规格”，则至少补足到**可审核的多规格形态**；当前执行口径为：**至少两种有效租用方式**。

### 1.4 图片与描述规则
1. **主图必须有**。
2. **描述图必须有**；不上传描述图，常见报错为：`商品描述需要包含图片`。
3. **商品描述无图片会被打回**。
4. 当前统一执行图片规范：
   - 白底 `#FFF`
   - 尺寸 `≥600×600`（推荐 `800×800`）

> 注：白底与尺寸来自已确认规范结论；若后续拿到更完整官方原文，再回填规则库。

### 1.5 表单填写顺序（平台侧约束）

```text
基础信息 → 销售规格 → 支付信息 → 物流信息 → 商品信息 → 提交审核
```

每步必须点“下一步”走到最后；跳步可能导致数据未保存。

## 2. 内部定价策略层（不是平台明文）

> 这一层是我方运营策略，用于统一出价，不等于平台审核规则。

### 2.1 定价计算

```bash
python3 skills/rrz-publish/scripts/rrz_pricing.py <设备价值>
```

| 项目 | 公式 |
|------|------|
| 基础月租 | 设备价值 × 15% |
| 押金 | 设备价值 × 80% |

### 2.2 租期阶梯

| 租期 | 计算 |
|------|------|
| 1天 | 月租/30 × 1.8 |
| 3天 | 月租/30 × 1.5 × 3 |
| 7天 | 月租 × 60% |
| 30天 | 月租 |
| 90天 | 月租×3 × 95折 |
| 180天 | 月租×6 × 9折 |
| 365天 | 月租×12 × 85折 |
| 730天 | 月租×24 × 80折 |
| 1095天 | 月租×36 × 75折 |

### 2.3 双方案
- 方案一：`租完归还` / 到期须归还 → 基础价格
- 方案二：`到期可归还/续租` → 基础价格 × 1.15

### 2.4 运费规则

| 租期总租金 | 寄送 | 归还 |
|-----------|------|------|
| ≤¥300 | ❌ | ❌ |
| ¥301-500 | ✅ | ❌ |
| ¥501-800 | ✅ | ❌ |
| >¥800 | ✅ | ✅ |

## 3. 执行实现规则（工程层，不是平台文案）

### 3.1 图片上传主路线
在 iframe 内注入 `rrz-e2e/scripts/rrz_upload.js`，用 SDK 直传 OSS：

```javascript
const {done, url} = await window.rrzUploadOne(base64String, 'jpg');
```

### 3.2 两个上传入口
- 主图：`.add-btn-wrap`
- 描述图：`.desc-add-btn-wrap`

### 3.3 写入 Vue 数据层
DOM 改值无效，必须写真实数据层；后端检查的描述图片字段是 `detail_picture` 映射链，不是只看 HTML 里的 `<img>`。

执行口径：
- 主图要写到主图字段
- 描述图要写到描述图字段
- 标题与核心字段写完后要读回验证

## 4. 发布前规则检查清单（必须过）

### A. 标题
- [ ] 标题含成色
- [ ] 标题含品牌
- [ ] 标题含具体型号
- [ ] 标题含具体使用场景
- [ ] 标题未出现禁表述

### B. 型号一致性
- [ ] 标题型号 = 描述型号 = 商品实际型号

### C. 套餐/规格
- [ ] 套餐名使用平台认可模板
- [ ] 至少两种有效租用方式/可审核规格
- [ ] 规格表头不是“套餐”

### D. 图片/描述
- [ ] 已上传主图
- [ ] 已上传描述图
- [ ] 商品描述满足“包含图片”要求
- [ ] 图片为白底，尺寸满足最低要求

### E. 禁表述
- [ ] 标题无 `租赁/出租/免押/免息/分期/最`
- [ ] 套餐/描述图文案无上述禁表述
- [ ] `测试` 已改成 `调试`

## 5. 常见驳回 → 修复动作

| 驳回/风险 | 修复动作 |
|---|---|
| 标题缺型号 | 补完整品牌+型号 |
| 标题与描述型号不一致 | 全量统一型号写法 |
| 套餐名缺租用方式 | 改为标准模板名称 |
| 规格表头写成“套餐” | 改为“型号/参数/配置” |
| 有“租赁”字眼 | 全量替换为“租用”或删除 |
| 商品描述无图片 | 通过描述图入口补图 |
| 套餐规格不足 | 补齐有效租用方式规格 |
